5  Sessions und To-Do-App

5.1 Das Problem: HTTP vergisst alles

Stell dir vor, du kaufst online ein: Du browsst ein Produkt an, legst es in den Warenkorb, navigierst woanders hin, und plötzlich ist dein Warenkorb leer. Frustrierend, richtig?

Das ist das HTTP-Problem: Das HyperText Transfer Protocol ist zustandslos (stateless). Jede Request ist für den Server eine neue Person – er erinnert sich nicht an die vorherige.

Die Armband-Analogie: Stell dir ein Schwimmbad vor: - Der Server ist das Personal - Jeder Besucher bekommt ein Armband (Cookie) mit einer Nummer - Der Server hat eine dicke Akte (Session) für jede Nummer - Mit dem Armband kann der Server sagen: “Aha, du bist Nummer 42 – dein Eintrag zeigt, dass du Platz 5 reserviert hast”

Sessions speichern Daten serverseitig und verknüpfen sie mit einem Cookie im Browser.

5.2 Sessions in Flask

Flask macht Sessions super einfach:

from flask import Flask, session

app = Flask(__name__)
app.secret_key = 'dein-super-geheimes-passwort-123!'  # Wichtig!
Warnung

Der secret_key muss gesetzt sein, sonst funktionieren Sessions nicht. Nutze in der Produktion einen zufälligen, starken String. Niemals ins Git committen!

5.2.1 Session-Operationen

# Wert speichern
session['username'] = 'alice'
session['farbe'] = 'blau'

# Wert auslesen
username = session.get('username', 'Gast')  # Default: 'Gast'

# Wert löschen
session.pop('username', None)

# Prüfen, ob Schlüssel existiert
if 'username' in session:
    print(f"Hallo, {session['username']}!")

# Alles löschen
session.clear()
Hinweis

session.get(schlüssel, default) ist sicherer als session[schlüssel], denn es crasht nicht, wenn der Schlüssel fehlt.

5.3 Login-System: Code-Along

Lass uns ein echtes Login-System bauen!

5.3.1 Schritt 1: Datenbank (vereinfacht)

Für dieses Beispiel nutzen wir eine gehärtete User-Liste:

from flask import Flask, render_template, request, redirect, url_for, session

app = Flask(__name__)
app.secret_key = 'mein-geheimnis-2024'

# Hardcodiert für Demo (in der Praxis: Datenbank + Passwort-Hashing!)
NUTZER = {
    'admin': 'geheim123',
    'schueler': 'passwort'
}

5.3.2 Schritt 2: Login-Route

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        username = request.form.get('username', '').strip()
        password = request.form.get('password', '')

        # Credentials checken
        if username in NUTZER and NUTZER[username] == password:
            session['username'] = username
            return redirect(url_for('startseite'))
        else:
            return render_template('login.html', fehler='Ungültige Anmeldedaten!')

    return render_template('login.html')

5.3.3 Schritt 3: Logout-Route

@app.route('/logout')
def logout():
    session.pop('username', None)
    return redirect(url_for('startseite'))

5.3.6 Schritt 6: Geschützte Routes

Nur angemeldete Nutzer dürfen rein:

@app.route('/dashboard')
def dashboard():
    if 'username' not in session:
        return redirect(url_for('login'))

    return render_template('dashboard.html')

@app.route('/admin')
def admin():
    if 'username' not in session:
        return redirect(url_for('login'))
    if session['username'] != 'admin':
        return "Zugriff verweigert!", 403

    return render_template('admin.html')

Immer checken: 'username' in session bevor du geschützte Daten zeigst!

5.4 Sessions im Browser untersuchen

Du kannst deine Sessions sehen:

  1. Öffne DevTools: F12
  2. Gehe zu Application (Chrome) oder Storage (Firefox)
  3. Links: Cookies
  4. Wähle deine Website
  5. Du siehst ein Cookie namens session – das ist verschlüsselt!
Tipp

Der Wert ist kryptographisch signiert. Das bedeutet: Nutzer können den Cookie-Wert nicht fälschen (Flask prüft die Signatur).

5.5 Übung 1: Besucherzähler

Schreibe eine Route /counter, die: - Zählt, wie oft du die Seite besucht hast (in der aktuellen Session) - Das Display zeigt: “Du hast diese Seite X mal besucht” - Es gibt einen Reset-Button, der den Zähler zurücksetzt

Hinweise: - Nutze session.get('visit_count', 0) um den aktuellen Wert zu holen - Speichere den neuen Wert: session['visit_count'] = ... - Der Reset-Button sendet ein POST-Request an eine separate Route - Denk dran: Der Zähler wird zurückgesetzt, wenn der Browser die Cookies löscht!

5.6 Übung 2: Zahlenratespiel (Challenge)

Baue ein Zahlenratespiel:

Das Spiel: - Der Server denkt sich eine Zahl zwischen 1 und 100 - Der Spieler rät immer wieder - Der Server sagt: “Zu hoch!”, “Zu tief!” oder “Richtig!” - Der Spieler sieht, wie viele Versuche er gebraucht hat - Es gibt einen Highscore (beste Versuche) in der Session

Anforderungen: - POST-Route für jeden Rateversucht - Session speichert: aktuelle Zahl, Versuchszähler, Highscore - Nach Sieg: Option “Neues Spiel” - Bootstrap-Styling für Statusmeldungen

5.7 Mini-Projekt: To-Do-App mit CRUD

Jetzt bauen wir eine richtige Web-App: Eine To-Do-Liste mit Datenbank!

5.7.1 Datenbank-Schema

Erstelle eine SQLite-Datenbank mit dieser Tabelle:

CREATE TABLE IF NOT EXISTS todos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    aufgabe TEXT NOT NULL,
    erledigt INTEGER DEFAULT 0,
    erstellt TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

erledigt speichert 0 (offen) oder 1 (erledigt). So können wir nach Status filtern.

5.7.2 Projekt-Struktur

todo-app/
├── app.py
├── database.db         (wird automatisch erstellt)
└── templates/
    ├── base.html
    ├── index.html
    └── form.html

5.7.3 Code-Gerüst

app.py (Anfang):

import sqlite3
from flask import Flask, render_template, request, redirect, url_for
from datetime import datetime

app = Flask(__name__)

def db_connection():
    """Stelle eine DB-Verbindung her"""
    conn = sqlite3.connect('database.db')
    conn.row_factory = sqlite3.Row
    return conn

def init_db():
    """Erstelle Tabelle, falls nicht vorhanden"""
    conn = db_connection()
    conn.execute('''
        CREATE TABLE IF NOT EXISTS todos (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            aufgabe TEXT NOT NULL,
            erledigt INTEGER DEFAULT 0,
            erstellt TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )
    ''')
    conn.commit()
    conn.close()

if __name__ == '__main__':
    init_db()
    app.run(debug=True)

5.7.4 READ: Alle Todos anzeigen

@app.route('/')
def index():
    filter_typ = request.args.get('filter', 'alle')  # 'alle', 'offen', 'erledigt'
    conn = db_connection()

    if filter_typ == 'offen':
        todos = conn.execute('SELECT * FROM todos WHERE erledigt = 0 ORDER BY erstellt DESC').fetchall()
    elif filter_typ == 'erledigt':
        todos = conn.execute('SELECT * FROM todos WHERE erledigt = 1 ORDER BY erstellt DESC').fetchall()
    else:
        todos = conn.execute('SELECT * FROM todos ORDER BY erledigt ASC, erstellt DESC').fetchall()

    # Statistiken
    stats = conn.execute('''
        SELECT
            COUNT(*) as total,
            SUM(CASE WHEN erledigt = 0 THEN 1 ELSE 0 END) as offen,
            SUM(CASE WHEN erledigt = 1 THEN 1 ELSE 0 END) as fertig
        FROM todos
    ''').fetchone()

    conn.close()
    return render_template('index.html', todos=todos, filter=filter_typ, stats=stats)

5.7.5 CREATE: Neues Todo hinzufügen

@app.route('/add', methods=['GET', 'POST'])
def add():
    if request.method == 'POST':
        aufgabe = request.form.get('aufgabe', '').strip()

        if not aufgabe:
            return render_template('form.html', fehler='Bitte gib eine Aufgabe ein!')

        if len(aufgabe) > 500:
            return render_template('form.html', fehler='Zu lang! Max. 500 Zeichen.')

        conn = db_connection()
        conn.execute('INSERT INTO todos (aufgabe) VALUES (?)', (aufgabe,))
        conn.commit()
        conn.close()

        return redirect(url_for('index'))

    return render_template('form.html')

5.7.6 UPDATE: Todo-Status togglen

@app.route('/toggle/<int:todo_id>')
def toggle(todo_id):
    conn = db_connection()
    todo = conn.execute('SELECT * FROM todos WHERE id = ?', (todo_id,)).fetchone()

    if todo:
        neuer_status = 1 - todo['erledigt']  # Toggle: 0->1 oder 1->0
        conn.execute('UPDATE todos SET erledigt = ? WHERE id = ?', (neuer_status, todo_id))
        conn.commit()

    conn.close()
    return redirect(url_for('index'))

5.7.7 DELETE: Todo löschen

@app.route('/delete/<int:todo_id>', methods=['POST'])
def delete(todo_id):
    conn = db_connection()
    conn.execute('DELETE FROM todos WHERE id = ?', (todo_id,))
    conn.commit()
    conn.close()
    return redirect(url_for('index'))
