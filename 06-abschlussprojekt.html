6  Abschlussprojekt

In diesem letzten Block bringst du alles zusammen! Du wirst eine eigene Web-App von Grund auf entwickeln und dabei all die Konzepte aus den Blöcken 1-5 einsetzen: Flask, Datenbanken, Templates, Formulare und mehr.

Tipp

Das Projekt ist dein Showcase – zeige, was du gelernt hast! Es soll nicht perfekt sein, sondern funktional und strukturiert.

6.1 Überblick über die Aufgabe

Zusammen mit einem Partner (oder allein, wenn du möchtest) entwickelst du eine vollständige Flask-Anwendung. Das Projekt ist in drei Sprints aufgeteilt:

  • Sprint 1 (UE 16): Grundgerüst aufbauen
  • Sprint 2 (UE 17): Features ausarbeiten und Feinschliff
  • UE 18: Präsentation und Abschlussreflexion

6.2 Anforderungen

6.2.1 Mindestanforderungen (MUSS)

Deine App muss folgende Dinge erfüllen, um bestanden zu werden:

  • Min. 3 verschiedene Routes (z.B. Startseite, Übersicht, Detail)
  • Min. 1 Formular mit POST-Request und Validierung
  • Min. 1 Datenbanktabelle
  • Template Inheritance (base.html mit Blöcken)
  • Bootstrap für ein anständiges Design
  • Flash Messages für Benutzer-Feedback
  • Parametrisierte SQL-Queries (um SQL-Injection zu vermeiden!)

6.2.2 SOLL-Anforderungen (für gute Note)

Wenn du diese Features einbaust, wird deine Note deutlich besser:

  • 2+ Datenbanktabellen mit Beziehungen
  • Alle CRUD-Operationen: Create, Read, Update, Delete
  • Sessions (z.B. für Login oder Favoriten)
  • Custom CSS (nicht nur Bootstrap)
  • Fehlerbehandlung (try-except, Custom Error Pages)

6.2.3 KANN-Anforderungen (Bonus-Punkte)

Zusätzliche Features für Überraschung und Glanz:

  • Suchfunktion
  • Benutzerrollen / Authentifizierung
  • Responsive Design (Mobile-friendly)
  • Kreatives UI-Design
  • Weitere Ideen von dir selbst!

6.3 Projekt-Ideen

Hier sind einige Ideen, sortiert nach Schwierigkeitsgrad. Such dir eine aus, die dir gefällt – oder bring deine eigene Idee mit!

6.3.1 ⭐⭐ Anfänger-Projekte

Rezeptsammlung

Nutzer können Rezepte speichern, bearbeiten und löschen. Jedes Rezept hat Titel, Zutaten, Anleitung und Schwierigkeitsgrad.

Datenbank: recipes (id, title, ingredients, instructions, difficulty, created_at)

Main Routes: /, /recipe/<id>, /add, /edit/<id>, /delete/<id>

Bonus: Kategorie-Filter, Suchfunktion, Druckversion

Klassen-Abstimmungstool

Deine Klasse kann Fragen abstimmen (z.B. “Wann treffen wir uns?”). Jedes Voting läuft für eine bestimmte Zeit.

Datenbank: polls (id, question, created_at, ends_at), votes (id, poll_id, option, count)

Main Routes: /, /poll/<id>, /create, /vote/<id>

Bonus: Live-Ergebnisse, Grafiken, anonyme Abstimmung

Bookmark-Manager

Speichere deine liebsten Websites mit Tags und Notizen. Schnell wiederfinden, leicht teilen.

Datenbank: bookmarks (id, url, title, notes, tags, created_at)

Main Routes: /, /bookmark/<id>, /add, /edit/<id>, /delete/<id>, /tag/<tag>

Bonus: Tag-Cloud, QR-Code generieren, Import/Export

6.3.2 ⭐⭐⭐ Mittelschwer

Quiz-App

Erstelle Quiz zu verschiedenen Themen. Nutzer können sich testen und ihre Punkte vergleichen.

Datenbank: quizzes (id, title, description), questions (id, quiz_id, question, answer_a, answer_b, answer_c, answer_d, correct), results (id, user, quiz_id, score, taken_at)

Main Routes: /, /quiz/<id>, /quiz/<id>/question/<num>, /results/<id>

Bonus: Leaderboard, Timer pro Frage, Schwierigkeitsstufen

Micro-Blog

Nutzer schreiben Posts, können diese kommentieren und liken.

Datenbank: posts (id, author, content, created_at, likes), comments (id, post_id, author, content, created_at)

Main Routes: /, /post/<id>, /create, /delete/<id>, /comment/<post_id>

Bonus: Nutzer-Profile, Hashtags, Likes-Counter, Sortierung

Vokabeltrainer

Lerne Vokabeln mit verschiedenen Modi (Abfrage, Schreiben, Multiple Choice).

Datenbank: vocabulary_sets (id, title, language_from, language_to), words (id, set_id, word_from, word_to, difficulty)

Main Routes: /, /set/<id>, /train/<id>, /results/<id>

Bonus: Lernstatistiken, Wiederholungs-Algorithmus (Spaced Repetition), Audio-Aussprache

6.3.3 ⭐⭐⭐⭐ Fortgeschritten

Schulplaner

Ein digitaler Planer für deine Klasse: Hausaufgaben, Klausurtermine, Vertretungsplan in einer App.

Datenbank: tasks (id, title, description, due_date, priority, user_id), exams (id, subject, date, room), substitutions (id, subject, date, original_teacher, substitute)

Main Routes: /, /tasks, /tasks/add, /exam-schedule, /substitutions, /user/<id>

Bonus: Benachrichtigungen, Kalender-View, Fächerfilter, Teamfunktion, Import aus ICS

Nicht deine Lieblings-Idee dabei? Kein Problem! Du kannst auch eine eigene Idee einbringen. Sprich mit deinem Kursleiter!

6.5.2 Schritt 3: Datenbank initialisieren

Erstelle eine init_db.py:

import sqlite3

conn = sqlite3.connect('database.db')
c = conn.cursor()

with open('schema.sql', 'r') as f:
    c.executescript(f.read())

conn.commit()
conn.close()
print("Datenbank erstellt!")

Dann ausführen: python init_db.py

6.5.4 Schritt 5: app.py mit Helper

Starte mit dieser Vorlage:

import sqlite3
from flask import Flask, render_template, request, flash, redirect, url_for, session

app = Flask(__name__)
app.secret_key = 'dein-geheimer-schluessel-change-this'

def get_db_connection():
    """Verbindung zur Datenbank öffnen."""
    conn = sqlite3.connect('database.db')
    conn.row_factory = sqlite3.Row
    return conn

@app.route('/')
def startseite():
    """Startseite anzeigen."""
    return render_template('startseite.html')

if __name__ == '__main__':
    app.run(debug=True)

6.5.5 Schritt 6-7: Erste Routes

Baue zwei einfache Routes:

  • Eine READ-Route (z.B. Übersichtsliste, GET)
  • Eine CREATE-Route (z.B. Formular + Speichern, GET + POST)

Beispiel READ:

@app.route('/rezepte')
def rezepte_liste():
    conn = get_db_connection()
    rezepte = conn.execute('SELECT * FROM rezepte').fetchall()
    conn.close()
    return render_template('rezepte.html', rezepte=rezepte)

Beispiel CREATE:

@app.route('/rezept/add', methods=['GET', 'POST'])
def rezept_add():
    if request.method == 'POST':
        titel = request.form['titel']

        # Validierung
        if not titel:
            flash('Titel ist erforderlich!', 'error')
            return redirect(url_for('rezept_add'))

        conn = get_db_connection()
        conn.execute('INSERT INTO rezepte (titel) VALUES (?)', (titel,))
        conn.commit()
        conn.close()

        flash('Rezept gespeichert!', 'success')
        return redirect(url_for('rezepte_liste'))

    return render_template('rezept_form.html')

6.6 Pair Programming in Sprint 1

Wenn du mit einem Partner arbeitest, nutzt Pair Programming: Ein Partner schreibt Code (Driver), der andere schaut zu und gibt Feedback (Navigator).

So wechselt ihr: - Alle 10 Minuten die Rollen tauschen - Der Navigator sitzt nicht nur rum – er schaut auf Fehler, schlägt Optimierungen vor - Beide müssen den Code verstehen, nicht nur tippen

Pair Programming ist anfangs etwas langsamer, aber danach habt ihr besseren Code und weniger Bugs. Vertraut dem Prozess!

6.7 Sprint 2: Features und Feinschliff (UE 17)

Sprint 2 ist für Vollständigkeit und Qualität. Nutze diese Prioritäts-Liste:

6.7.1 Top 5 Debugging-Tipps

Wenn etwas nicht funktioniert:

  1. Jinja2 Syntax-Fehler – Prüfe deine { } und {% %}

    ❌ {{ name }} wird das Wort nicht in Anführungszeichen
✅ {{ name }} sollte einfach so funktionieren

  2. OperationalError – Meist ist eine SQL-Spalte falsch geschrieben

    No such column: titel (aber du schreibst `titel`?)
→ Check deine schema.sql und deine Query!

  3. 404 Fehler – Route existiert nicht

    @app.route('/rezepte')  # Route heißt /rezepte
# aber du besuchst /rezepte-liste? Nicht gefunden!

  4. 405 Fehler – Falsche HTTP-Methode

    @app.route('/rezept/add', methods=['GET', 'POST'])
# Wenn du mit DELETE fragst, wird 405 zurückgegeben

  5. Datenbank-Änderungen nicht gespeichert – Vergessenes commit()

    conn.execute('INSERT ...')
conn.commit()  # ← NICHT VERGESSEN!
conn.close()

6.7.2 Prioritäts-Liste für Sprint 2

Muss erledigt sein: 1. Alle CRUD-Operationen (Create, Read, Update, Delete) 2. Formular-Validierung 3. Flash Messages für Feedback 4. Fehlerbehandlung (try-except)

Sollte erledigt sein: 5. Bootstrap + eigenes CSS 6. Code-Kommentare 7. Getestete Edge-Cases (z.B. leere Eingaben)

Kann zusätzlich: 8. Suchfunktion 9. Sessions / Authentifizierung 10. Bonus-Features

6.7.3 Code-Kommentar Template

Nutze diesen Style für deine Funktionen:

@app.route('/rezept/<id>')
def rezept_details(id):
    """
    Zeigt ein einzelnes Rezept mit Titel, Zutaten und Anleitung.

    Args:
        id: Die ID des Rezepts aus der URL

    Returns:
        HTML der Rezept-Detailseite oder 404
    """
    conn = get_db_connection()
    rezept = conn.execute('SELECT * FROM rezepte WHERE id = ?', (id,)).fetchone()
    conn.close()

    if rezept is None:
        abort(404)

    return render_template('rezept_details.html', rezept=rezept)
