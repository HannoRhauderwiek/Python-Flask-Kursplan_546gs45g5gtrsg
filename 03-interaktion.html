3  Interaktion

Bis jetzt haben wir statische HTML-Seiten mit Flask ausgeliefert. Jetzt wird es interaktiv! In diesem Kapitel lernst du, wie du mit Nutzereingaben umgehen kannst, wie du Templates clever wiederverwendest und wie du mit deinen Userinnen kommunizierst.

3.1 Template-Vererbung: DRY-Prinzip

Stell dir vor, du möchtest auf jeder Seite einer Website die gleiche Navigation, den gleichen Footer und die gleiche CSS-Datei haben. Würdest du das in jede HTML-Datei kopieren? Das wäre furchtbar! Der Code würde sich überall duplizieren und bei Änderungen müsstest du alles anfassen.

Das DRY-Prinzip (Don’t Repeat Yourself) sagt: Schreib deinen Code nur einmal auf. Flask hat dafür Template-Vererbung – vergleichbar mit Briefpapier:

┌─────────────────────────────────┐
│        BASE-TEMPLATE            │
│  ┌───────────────────────────┐  │
│  │ Kopfzeile (Navigation)    │  │
│  ├───────────────────────────┤  │
│  │  {% block content %}      │  │
│  │  [Platzhalter]           │  │
│  │  {% endblock %}           │  │
│  ├───────────────────────────┤  │
│  │ Fußzeile (Footer)         │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘

    ↓ EXTENDS

┌─────────────────────────────────┐
│    CHILD-TEMPLATE (index.html)  │
│  Spezifischer Inhalt für Block  │
└─────────────────────────────────┘

3.1.1 Code-Along: Base-Template

Erstelle templates/base.html:

<!DOCTYPE html>
<html lang="de">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{% block title %}Meine Flask-App{% endblock %}</title>
    <!-- Bootstrap CDN -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
    <link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}">
</head>
<body>
    <nav class="navbar navbar-expand-lg navbar-dark bg-dark">
        <div class="container">
            <a class="navbar-brand" href="/">Meine App</a>
            <button class="navbar-toggler" type="button" data-bs-toggle="collapse"
                    data-bs-target="#navbarNav">
                <span class="navbar-toggler-icon"></span>
            </button>
            <div class="collapse navbar-collapse" id="navbarNav">
                <ul class="navbar-nav ms-auto">
                    <li class="nav-item"><a class="nav-link" href="/">Home</a></li>
                    <li class="nav-item"><a class="nav-link" href="/kontakt">Kontakt</a></li>
                </ul>
            </div>
        </div>
    </nav>

    <main class="container mt-5">
        {% with messages = get_flashed_messages(with_categories=true) %}
            {% if messages %}
                {% for category, message in messages %}
                    <div class="alert alert-{{ category }} alert-dismissible fade show" role="alert">
                        {{ message }}
                        <button type="button" class="btn-close" data-bs-dismiss="alert"></button>
                    </div>
                {% endfor %}
            {% endif %}
        {% endwith %}

        {% block content %}{% endblock %}
    </main>

    <footer class="mt-5 py-4 bg-light text-center">
        <p>&copy; 2026 Meine Flask-App</p>
    </footer>

    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

3.1.2 Code-Along: Child-Template

Erstelle templates/index.html:

{% extends "base.html" %}

{% block title %}Startseite{% endblock %}

{% block content %}
<h1>Willkommen!</h1>
<p>Das ist deine Startseite mit Base-Template-Vererbung.</p>
{% endblock %}

Erkennst du das Muster? Die Child-Template sagt: “Nutze base.html als Grundlage und fülle nur meinen Content-Block!”

url_for() ist magisch: Statt href="/static/style.css" schreibst du href="{{ url_for('static', filename='style.css') }}". Flask findet die Datei automatisch – auch wenn du deine URL-Struktur änderst. Immer verwenden!

3.2 Formulare und Nutzereingaben

Jetzt verbinden wir HTML-Formulare (Block 1) mit Flask-Verarbeitung. Der Prozess:

┌──────────────┐
│  HTML-Form   │
│  <form>      │
└──────┬───────┘
       │ POST/GET
       ↓
┌──────────────────────────────────────┐
│  Flask-Route: request.method prüfen  │
│  request.form['feldname'] auslesen   │
│  Daten verarbeiten                   │
└──────────────────────────────────────┘
       │
       ↓
┌──────────────────────────┐
│  Antwort an User         │
│  HTML rendern oder       │
│  Umleitung               │
└──────────────────────────┘

3.2.1 GET vs POST

  • GET = “Gucken” – Der User fragt nach Daten. Parameter sichtbar in der URL: ?name=Max&alter=25
  • POST = “Paket schicken” – Der User schickt Daten (sicher, versteckt). Besser für Formulare!

3.2.2 Code-Along: Namensgenerator

templates/namens_form.html:

{% extends "base.html" %}

{% block title %}Namensgenerator{% endblock %}

{% block content %}
<h2>Dein Spitzname-Generator</h2>

<form method="POST" action="/namensgenerator">
    <div class="mb-3">
        <label for="vorname" class="form-label">Dein Vorname:</label>
        <input type="text" class="form-control" id="vorname" name="vorname" required>
    </div>
    <button type="submit" class="btn btn-primary">Spitzname generieren</button>
</form>

{% if ergebnis %}
<div class="alert alert-success mt-4">
    <h4>Dein Spitzname:</h4>
    <p><strong>{{ ergebnis }}</strong></p>
</div>
{% endif %}
{% endblock %}

app.py:

from flask import Flask, render_template, request

app = Flask(__name__)
app.secret_key = 'dein-geheimes-schlüsselchen'

@app.route('/namensgenerator', methods=['GET', 'POST'])
def namensgenerator():
    ergebnis = None

    if request.method == 'POST':
        vorname = request.form['vorname']
        # Spitzname: erste 3 Buchstaben + "o"
        ergebnis = vorname[:3].lower() + 'o'

    return render_template('namens_form.html', ergebnis=ergebnis)

if __name__ == '__main__':
    app.run(debug=True)

Taschenrechner-Formular

Erstelle eine Route /rechner, die: - Ein HTML-Formular mit zwei Zahlen-Input-Feldern (zahl1, zahl2) - Ein Dropdown (operation) mit Optionen: +, -, *, / - Nach dem Absenden wird POST verarbeitet und das Ergebnis angezeigt

Tipp: request.form.get() hilft dir, auf Formularfelder zuzugreifen. Du brauchst auch Python-Logik zum Rechnen!

3.3 Flash-Nachrichten und Validierung

Flash-Nachrichten sind temporäre Mitteilungen an deinen User: “Danke, dass du dich angemeldet hast!” oder “Fehler: E-Mail ungültig!”. Sie erscheinen einmal und verschwinden dann.

3.3.1 Code-Along: Feedback-Formular

templates/feedback.html:

{% extends "base.html" %}

{% block title %}Feedback{% endblock %}

{% block content %}
<h2>Dein Feedback</h2>

<form method="POST" action="/feedback">
    <div class="mb-3">
        <label for="name" class="form-label">Name:</label>
        <input type="text" class="form-control" id="name" name="name" required>
    </div>

    <div class="mb-3">
        <label for="email" class="form-label">E-Mail:</label>
        <input type="email" class="form-control" id="email" name="email" required>
    </div>

    <div class="mb-3">
        <label for="nachricht" class="form-label">Nachricht:</label>
        <textarea class="form-control" id="nachricht" name="nachricht" rows="4" required></textarea>
    </div>

    <button type="submit" class="btn btn-success">Absenden</button>
</form>

<h3 class="mt-5">Alle Feedbacks:</h3>
{% if feedbacks %}
    <ul class="list-group">
        {% for feedback in feedbacks %}
            <li class="list-group-item">
                <strong>{{ feedback.name }}</strong> ({{ feedback.email }})<br>
                {{ feedback.nachricht }}
            </li>
        {% endfor %}
    </ul>
{% else %}
    <p class="text-muted">Noch kein Feedback eingegangen.</p>
{% endif %}
{% endblock %}

app.py (ergänzt):

from flask import Flask, render_template, request, redirect, url_for, flash

app = Flask(__name__)
app.secret_key = 'dein-geheimes-schlüsselchen'

# Globale Liste (wird bei Neustart gelöscht – später Datenbank!)
feedbacks = []

def ist_email_gueltig(email):
    """Einfache E-Mail-Validierung"""
    return '@' in email and '.' in email

@app.route('/feedback', methods=['GET', 'POST'])
def feedback():
    if request.method == 'POST':
        name = request.form.get('name', '').strip()
        email = request.form.get('email', '').strip()
        nachricht = request.form.get('nachricht', '').strip()

        # Validierung
        if not name:
            flash('Bitte fülle deinen Namen ein!', 'danger')
            return render_template('feedback.html', feedbacks=feedbacks)

        if not ist_email_gueltig(email):
            flash('E-Mail ungültig! Nutze das Format name@beispiel.de', 'warning')
            return render_template('feedback.html', feedbacks=feedbacks)

        if len(nachricht) < 5:
            flash('Deine Nachricht muss mindestens 5 Zeichen lang sein.', 'danger')
            return render_template('feedback.html', feedbacks=feedbacks)

        # Alles OK – speichern und weiterleiten
        feedbacks.append({
            'name': name,
            'email': email,
            'nachricht': nachricht
        })

        flash('Danke für dein Feedback! 🎉', 'success')
        return redirect(url_for('feedback'))

    return render_template('feedback.html', feedbacks=feedbacks)

if __name__ == '__main__':
    app.run(debug=True)

Post/Redirect/Get (PRG)-Muster: Nach einem erfolgreichen POST sendest du ein redirect() statt render_template(). Warum? Wenn der User F5 drückt, wird die Form nicht nochmal abgesendet – die Nachricht wird nicht dupliziert. Clever!

3.3.2 Flash-Kategorien

Mit flash('text', 'kategorie') kannst du verschiedene Typen unterscheiden: - 'success' → grüne Nachricht ✓ - 'danger' → rote Nachricht ✗ - 'warning' → gelbe Nachricht ⚠ - 'info' → blaue Nachricht ℹ

In base.html werden sie automatisch mit Bootstrap-CSS gestylt!

Tipp

Eingabe-Validierung ist wichtig: Vertrau nie blind auf User-Eingaben! Prüfe immer auf: - Leere Felder - Gültige E-Mail-Formate - Mindestlängen - Zahlen statt Text (bei Zahlenfeldern)

Login mit Passwort-Challenge

Erstelle eine Route /login, die: - Ein HTML-Formular mit Benutzernamen und Passwort-Feld hat - Nach dem Absenden das Passwort mit einem hardcodierten Wert (z.B. "secret123") vergleicht - Bei richtigem Passwort: Flash-Nachricht “Willkommen!” + Umleitung zur Startseite - Bei falschem Passwort: Flash-Nachricht “Falsches Passwort!” + Rückkehr zum Formular

(Diese Challenge ist eine Vorschau auf Sitzungen in Block 5. Hier speichern wir noch keine Login-Info!)

Erweiterte Feedback-Sammlung

Füge deinem Feedback-System hinzu: - Einen Schwierigkeitsgrad (Dropdown: “Einfach”, “Mittel”, “Komplex”) - Ein Häkchen “Ich möchte kontaktiert werden” - Validierung: Der Schwierigkeitsgrad darf nicht leer sein - In der Feedback-Liste: Zeige auch den Schwierigkeitsgrad und das Häkchen

Nutze die neuen Felder im Formular und verarbeite sie in der Flask-Route!

3.4 Zusammenfassung

Template-Vererbung:
├─ base.html (Grundlayout, Navigation, Footer)
└─ child.html ({% extends %}, spezifischer Inhalt)

Formulare verarbeiten:
├─ HTML <form method="POST">
├─ Flask @app.route(..., methods=['GET', 'POST'])
└─ request.form['feldname'] oder request.args.get('param')

Flash-Nachrichten:
├─ app.secret_key setzen (WICHTIG!)
├─ flash('text', 'kategorie')
├─ get_flashed_messages(with_categories=true) in Template
└─ PRG-Muster: Bei Erfolg redirect(), bei Fehler render_template()

Validierung:
├─ Leere Felder prüfen
├─ Format prüfen (E-Mail, Zahl, etc.)
├─ Länge/Größe prüfen
└─ Aussagekräftige Flash-Nachrichten zurückgeben
Warnung

Sicherheitshinweis: In diesem Block speichern wir Daten noch in Python-Listen – die verschwinden nach jedem Neustart! In Block 4 lernen wir Datenbanken kennen. Vertrau dich noch nicht mit echten Nutzerdaten an! 😉

Du bist jetzt bereit, interaktive Webanwendungen zu bauen! Mit Template-Vererbung hältst du deinen Code trocken, mit Formularen sammelst du Eingaben, und mit Flash-Nachrichten kommunizierst du mit deinen Userinnen. Los geht’s!
