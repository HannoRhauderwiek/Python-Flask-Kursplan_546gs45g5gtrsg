4  Daten speichern

4.1 Das Problem mit Listen

Stell dir vor: Du hast eine Anwendung, die eine Liste mit Büchern verwaltet. Du fügst Bücher hinzu, löscht sie, änderst sie – alles funktioniert perfekt. Aber dann startest du dein Programm neu… und puff: alle Daten sind weg!

buecher = []  # Die Daten sind nur im RAM – sobald das Programm endet: weg!

Das ist das Kernproblem: Python-Listen existieren nur im Arbeitsspeicher (RAM). Wenn das Programm beendet wird, sind alle Daten verloren.

Eine Datenbank ist ein persistentes Speichersystem. Die Daten bleiben erhalten, auch wenn das Programm neu gestartet wird.

4.2 Datenbanken vs. Listen

Vergleich:

Eigenschaft Python-Liste Datenbank
Speicherort RAM (Arbeitsspeicher) Festplatte / Datei
Datensicherheit Verloren nach Programmende Persistent, bleibt erhalten
Struktur Ungeordnet, flexibel Strukturiert (Tabellen)
Suchmöglichkeiten Schleifen nötig SQL-Abfragen (mächtig!)
Größe Begrenzt (RAM-Limit) Theoretisch unbegrenzt
Tipp

Für kleine Anwendungen wie ein Gästebuch oder eine Bücherverwaltung ist SQLite perfekt: eine komplette Datenbank in einer einzigen Datei, ohne Server-Installation nötig!

4.3 SQLite – Die Datenbank für Anfänger

SQLite ist ein “zero-configuration” Datenbanksystem. Kein Login, kein Server, kein Konfigurationswirrwarr – nur eine .db-Datei.

Das Beste: SQLite ist in Python eingebaut!

import sqlite3

4.3.1 Verbindung zur Datenbank

# Verbindung erstellen (oder Datei erzeugen, falls nicht existent)
conn = sqlite3.connect('buecher.db')

# Befehle ausführen
cursor = conn.cursor()

# Änderungen speichern
conn.commit()

# Fertig
conn.close()
Warnung

Vergiss conn.commit() nicht! Ohne diesen Befehl werden deine Änderungen nicht gespeichert!

4.3.2 Einfacherer Zugriff: Row-Factory

Normalerweise geben SQL-Abfragen Tupel zurück: (1, 'Harry Potter', 'J.K. Rowling'). Das ist unlesbar!

Mit row_factory kannst du auf Spalten wie ein Dictionary zugreifen:

import sqlite3

conn = sqlite3.connect('buecher.db')
conn.row_factory = sqlite3.Row  # ← Zauberei!
cursor = conn.cursor()

# Jetzt kann ich auf Spalten-Namen zugreifen:
cursor.execute("SELECT * FROM buecher")
for row in cursor:
    print(f"{row['titel']} von {row['autor']}")

conn.close()

4.4 SQL-Grundlagen mit SQLite

4.4.1 CREATE TABLE – Tisch vorbereiten

Eine Tabelle ist wie ein Excel-Sheet mit Spalten und Zeilen.

import sqlite3

conn = sqlite3.connect('buecher.db')
cursor = conn.cursor()

# Tabelle erstellen
cursor.execute('''
    CREATE TABLE IF NOT EXISTS buecher (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        titel TEXT NOT NULL,
        autor TEXT NOT NULL,
        jahr INTEGER,
        bewertung REAL
    )
''')

conn.commit()
conn.close()

Was bedeutet was?

  • id INTEGER PRIMARY KEY AUTOINCREMENT: Eindeutige Nummer, automatisch hochzählend
  • TEXT NOT NULL: Text-Feld, darf nicht leer sein
  • REAL: Dezimalzahl (z.B. für Bewertungen 1.0–5.0)
  • IF NOT EXISTS: Nur erstellen, falls noch nicht existent (verhindert Fehler)

Schau dir die Spalten-Definitionen an: Welche Datentypen brauchst du für (a) Namen, (b) Alter, (c) E-Mail-Adresse?

4.4.2 INSERT – Daten hinzufügen

import sqlite3

conn = sqlite3.connect('buecher.db')
cursor = conn.cursor()

# Einzel-Insert
cursor.execute('''
    INSERT INTO buecher (titel, autor, jahr, bewertung)
    VALUES (?, ?, ?, ?)
''', ('Harry Potter 1', 'J.K. Rowling', 1998, 4.8))

# Mehrere auf einmal
buecher_liste = [
    ('Der Herr der Ringe', 'J.R.R. Tolkien', 1954, 4.9),
    ('1984', 'George Orwell', 1949, 4.5),
]
cursor.executemany(
    'INSERT INTO buecher (titel, autor, jahr, bewertung) VALUES (?, ?, ?, ?)',
    buecher_liste
)

conn.commit()
conn.close()

WICHTIG: Immer ? als Platzhalter verwenden!

# ✅ Richtig
cursor.execute("SELECT * FROM buecher WHERE titel = ?", (search,))

# ❌ FALSCH – SQL-Injection Sicherheitslücke!
cursor.execute(f"SELECT * FROM buecher WHERE titel = '{search}'")
Warnung

SQL-Injection: Mit f-Strings kannst du Benutzer-Input direkt in SQL einbauen. Ein böser User könnte dann '; DROP TABLE buecher; -- eingeben und deine ganze Tabelle löschen! Immer ? verwenden!

4.4.3 SELECT – Daten auslesen

import sqlite3

conn = sqlite3.connect('buecher.db')
conn.row_factory = sqlite3.Row
cursor = conn.cursor()

# Alle Daten
cursor.execute("SELECT * FROM buecher")
alle = cursor.fetchall()
for buch in alle:
    print(f"{buch['titel']} ({buch['jahr']})")

# Ein einzelnes Ergebnis
cursor.execute("SELECT * FROM buecher WHERE id = ?", (1,))
buch = cursor.fetchone()
print(buch['titel'])

# Mit WHERE und ORDER BY
cursor.execute("""
    SELECT * FROM buecher
    WHERE bewertung > ?
    ORDER BY bewertung DESC
    LIMIT 5
""", (4.0,))
top_buecher = cursor.fetchall()

# COUNT und AVG
cursor.execute("SELECT COUNT(*) as anzahl FROM buecher")
print(cursor.fetchone()['anzahl'])

cursor.execute("SELECT AVG(bewertung) as durchschnitt FROM buecher")
print(cursor.fetchone()['durchschnitt'])

conn.close()

4.4.4 UPDATE – Daten ändern

cursor.execute("""
    UPDATE buecher
    SET bewertung = ?
    WHERE id = ?
""", (4.7, 1))

conn.commit()

4.4.5 DELETE – Daten löschen

cursor.execute("DELETE FROM buecher WHERE id = ?", (2,))
conn.commit()

4.5 CRUD – Die vier Säulen der Datenbank

Jede Datenbank-Operation ist eine dieser vier:

  • Create: INSERT INTO – Neue Einträge hinzufügen
  • Read: SELECT – Daten auslesen
  • Update: UPDATE – Bestehende Daten ändern
  • Delete: DELETE – Einträge löschen

Bücherverwaltung (reines Python, kein Flask)

Schreib ein Programm, das: 1. Eine buecher.db erstellt mit den Spalten: id, titel, autor, jahr 2. Ein Menü zeigt mit Optionen: - 1 – Buch hinzufügen (INSERT) - 2 – Alle Bücher zeigen (SELECT) - 3 – Buch aktualisieren (UPDATE) nach ID - 4 – Buch löschen (DELETE) nach ID - 5 – Beenden

Achte darauf, dass die Daten nach Programmende noch vorhanden sind!

4.6 Flask trifft SQLite

Jetzt wird’s richtig cool: Wir verbinden Flask mit SQLite und bauen eine richtige Web-Anwendung!

4.6.1 Das Schema: schema.sql

Für größere Projekte ist es üblich, die Tabellen-Struktur in eine separate schema.sql zu schreiben:

-- schema.sql
DROP TABLE IF EXISTS buecher;

CREATE TABLE buecher (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  titel TEXT NOT NULL,
  autor TEXT NOT NULL,
  jahr INTEGER,
  bewertung REAL DEFAULT 0.0
);

4.6.2 Init-Script: init_db.py

# init_db.py
import sqlite3

def init_db():
    """Initialisiert die Datenbank mit schema.sql"""
    conn = sqlite3.connect('instance/buecher.db')
    with open('schema.sql') as f:
        conn.executescript(f.read())
    conn.commit()
    conn.close()
    print("✓ Datenbank initialisiert!")

if __name__ == '__main__':
    init_db()

Dann einfach ausführen:

python init_db.py

4.6.3 Helper-Funktion: get_db_connection()

Um nicht überall sqlite3.connect() zu schreiben (DRY-Prinzip!), eine Hilfsfunktion:

# app.py
import sqlite3
from flask import Flask

app = Flask(__name__)

def get_db_connection():
    """Gibt eine Datenbankverbindung mit Row-Factory zurück"""
    conn = sqlite3.connect('instance/buecher.db')
    conn.row_factory = sqlite3.Row
    return conn

4.6.4 READ: Daten aus der DB in Template

@app.route('/')
def index():
    """Zeigt alle Bücher"""
    conn = get_db_connection()
    buecher = conn.execute('SELECT * FROM buecher ORDER BY id DESC').fetchall()
    conn.close()
    return render_template('buecher.html', buecher=buecher)

Template (templates/buecher.html):

<h1>Bücherverwaltung</h1>
<a href="{{ url_for('neues_buch') }}">+ Buch hinzufügen</a>

<table>
  <thead>
    <tr>
      <th>ID</th>
      <th>Titel</th>
      <th>Autor</th>
      <th>Jahr</th>
      <th>Aktionen</th>
    </tr>
  </thead>
  <tbody>
    {% for buch in buecher %}
    <tr>
      <td>{{ buch['id'] }}</td>
      <td>{{ buch['titel'] }}</td>
      <td>{{ buch['autor'] }}</td>
      <td>{{ buch['jahr'] }}</td>
      <td>
        <a href="{{ url_for('loeschen', id=buch['id']) }}">Löschen</a>
      </td>
    </tr>
    {% endfor %}
  </tbody>
</table>

4.6.5 CREATE: Formular → INSERT

Route:

from flask import request, redirect, url_for, render_template

@app.route('/neu', methods=['GET', 'POST'])
def neues_buch():
    if request.method == 'POST':
        titel = request.form['titel']
        autor = request.form['autor']
        jahr = request.form['jahr']

        conn = get_db_connection()
        conn.execute(
            'INSERT INTO buecher (titel, autor, jahr) VALUES (?, ?, ?)',
            (titel, autor, jahr)
        )
        conn.commit()
        conn.close()

        return redirect(url_for('index'))

    return render_template('buch_hinzufuegen.html')

Template (templates/buch_hinzufuegen.html):

<h2>Buch hinzufügen</h2>
<form method="POST">
  <label>Titel:
    <input type="text" name="titel" required>
  </label>

  <label>Autor:
    <input type="text" name="autor" required>
  </label>

  <label>Jahr:
    <input type="number" name="jahr">
  </label>

  <button type="submit">Speichern</button>
  <a href="{{ url_for('index') }}">Abbrechen</a>
</form>
Tipp

Das POST-Redirect-GET Pattern (PRG) verhindert, dass ein User versehentlich das Formular zweimal absendet, wenn er F5 drückt!

4.6.6 DELETE: Eintrag löschen

@app.route('/loeschen/<int:id>')
def loeschen(id):
    conn = get_db_connection()
    conn.execute('DELETE FROM buecher WHERE id = ?', (id,))
    conn.commit()
    conn.close()
    return redirect(url_for('index'))
Warnung

Sicherheit: Echte Anwendungen sollten mit DELETE vorsichtiger umgehen (z.B. nur nach Bestätigung). Für Übungszwecke ist dies ok.

4.6.7 Beweis: Daten bleiben erhalten!

Starte deine Flask-App, füge ein paar Bücher hinzu, beende die App mit Ctrl+C, starte sie neu – die Bücher sind noch da!

Das ist der Unterschied zwischen Datenbanken und Listen.

Datenfluss in einer DB-gestützten Flask-App:

  1. Browser sendet Request (Klick auf Link oder Formular-Submit)
  2. Flask-Route wird ausgeführt
  3. Datenbank wird abgefragt oder aktualisiert
  4. Ergebnis wird ins Template eingefügt
  5. HTML wird zum Browser zurückgesendet
  6. Browser zeigt die Seite an

4.7 Mini-Projekt: Gästebuch

Nun bist du an der Reihe! Baue selbstständig ein Gästebuch mit Flask und SQLite.

4.7.1 Anforderungen

  • Schema: Eine Tabelle gaestebuch_eintraege mit:
    • id (Autoincrement)
    • name (Text, nicht null)
    • nachricht (Text, nicht null)
    • erstellt_am (Timestamp, Standardwert: jetzt)
  • Routes:
    • GET / – Zeigt alle Einträge (neueste zuerst)
    • GET /neu – Formular zum Erstellen
    • POST /neu – Verarbeitet Formular (INSERT)
  • Validierung:
    • Name und Nachricht dürfen nicht leer sein
    • Zeige Fehlermeldungen mit flash()
  • Templates:
    • index.html – Liste aller Einträge
    • neu.html – Formular
  • Design: Nutze Bootstrap für ein anständiges Aussehen

Tipp 1: Timestamp

In SQLite kannst du eine Spalte so definieren:

erstellt_am DATETIME DEFAULT CURRENT_TIMESTAMP

Oder in Python mit datetime:

from datetime import datetime
now = datetime.now().strftime('%Y-%m-%d %H:%M:%S')
conn.execute('INSERT INTO ... VALUES (..., ?)', (..., now))

Tipp 2: Struktur-Vorlage

gaestebuch/
├── app.py
├── schema.sql
├── init_db.py
├── instance/
│   └── gaestebuch.db
└── templates/
    ├── base.html
    ├── index.html
    └── neu.html

Tipp 3: Flash-Messages

from flask import flash

if not name or not nachricht:
    flash('Name und Nachricht sind erforderlich!', 'error')
    return redirect(url_for('neu'))

Im Template:

{% with messages = get_flashed_messages(with_categories=true) %}
  {% if messages %}
    {% for category, message in messages %}
      <div class="alert alert-{{ category }}">{{ message }}</div>
    {% endfor %}
  {% endif %}
{% endwith %}

4.7.2 Bonus-Aufgaben (freiwillig)

  • Delete: Füge einen “Löschen”-Button hinzu (mit <int:id> Route)
  • Search: Ermögliche Suche nach Namen oder Nachricht
  • Pagination: Zeige nur 10 Einträge pro Seite
  • Edit: Benutzer können eigene Einträge editieren
  • Like-System: Einträge können “geliked” werden (neue Spalte likes INTEGER)

Implementiere das Gästebuch nach den Anforderungen. Teste: 1. Füge mehrere Einträge hinzu 2. Beende die Flask-App und starte sie neu 3. Kontrolliere, dass die Einträge noch da sind ✓

Wenn du Zeit hast: Implementiere eine der Bonus-Aufgaben!

4.8 Zusammenfassung

Konzept Beispiel
Datenbank erstellen sqlite3.connect('datei.db')
Tabelle definieren CREATE TABLE IF NOT EXISTS ...
Daten einfügen INSERT INTO ... VALUES (?, ?, ?)
Daten auslesen SELECT * FROM ... WHERE ... ORDER BY ...
Daten ändern UPDATE ... SET ... WHERE ...
Daten löschen DELETE FROM ... WHERE ...
In Flask nutzen Helper-Funktion get_db_connection()
Sicherheit IMMER ? Platzhalter, NIEMALS f-Strings!
Hinweis

Glückwunsch! Du kannst nun Daten persistent speichern. Das ist ein riesiger Schritt: Deine Anwendungen können jetzt echte Daten verwalten, die über Neustarts hinweg bestehen bleiben.
