1  Das Web verstehen

Bevor wir Flask programmieren, müssen wir verstehen, wie das Web eigentlich funktioniert. Das mag zuerst theoretisch klingen – aber du wirst sehen: Wenn du weißt, wie Browser und Server miteinander sprechen, wird alles andere viel verständlicher!

1.1 UE 1: Wie funktioniert das Web?

1.1.1 Das Client-Server-Modell

Stell dir ein Restaurant vor:

┌─────────────────────────────────────────────┐
│  Du (Browser / Client) sitzt am Tisch      │
│                                             │
│  → "Ich möchte ein Schnitzel bestellen"   │
│                                             │
│  ← Und hier kommt das Schnitzel an!        │
│                                             │
└─────────────────────────────────────────────┘
         ↑            ↓
         │   Bestellung/Essen
         │
┌─────────────────────────────────────────────┐
│  Die Küche (Server)                        │
│                                             │
│  → "Alles klar, ich koche..."             │
│  ← "Fertig!"                               │
└─────────────────────────────────────────────┘

Im Web funktioniert das genau so:

  • Client = Dein Browser (Chrome, Firefox, Safari)
  • Server = Ein Computer irgendwo auf der Welt, der Websites speichert
  • Bestellung = HTTP-Request (deine Anfrage an den Server)
  • Essen = HTTP-Response (die Antwort vom Server mit der Website)
Hinweis

HTTP steht für “HyperText Transfer Protocol” – das ist einfach die Sprache, in der Browser und Server miteinander reden.

1.1.2 HTTP: Anfragen und Antworten

Wenn du eine Website aufrufst, passiert folgendes:

Dein Browser (Client) sendet einen HTTP-Request:

GET /index.html HTTP/1.1
Host: www.example.com

Das bedeutet: “Gib mir bitte die Datei index.html vom Server example.com”

Der Server antwortet mit einem HTTP-Response:

HTTP/1.1 200 OK
Content-Type: text/html

<html>
  <body>
    <h1>Willkommen!</h1>
  </body>
</html>

Das bedeutet: “OK, alles gut! Hier ist deine Website!”

1.1.3 GET und POST – Die zwei wichtigsten Methoden

Es gibt verschiedene Arten, mit einem Server zu kommunizieren. Die zwei wichtigsten sind:

GET-Request: - Du möchtest Daten vom Server holen (lesen) - Die Parameter sieht man in der URL: www.google.com/search?q=python - GET ist für Anfragen gedacht, die den Server nicht verändern

POST-Request: - Du möchtest Daten zum Server schicken (schreiben) - Beispiel: Ein Formular ausfüllen und absenden - Die Daten sind nicht in der URL sichtbar (sicherer für Passwörter!)

Tipp

Merksatz: GET = holen (lesen), POST = schicken (schreiben)

1.1.4 URLs verstehen

Eine URL hat mehrere Teile:

https://www.youtube.com/watch?v=dQw4w9WgXcQ&t=10s
│       │   │             │         │      │
│       │   │             │         │      └─ Parameter: Start bei 10 Sekunden
│       │   │             │         └──────── Parameter: Video-ID
│       │   │             └─────────────────── Query String (Parameter)
│       │   └──────────────────────────────── Path (Pfad)
│       └──────────────────────────────────── Domain (Server)
└──────────────────────────────────────────── Protokoll (https)
  • Protokoll (https://): Verschlüsselte Verbindung
  • Domain (www.youtube.com): Der Server
  • Path (/watch): Der Pfad auf dem Server
  • Parameter (?v=...&t=...): Zusätzliche Informationen

1.1.5 HTTP-Status-Codes

Der Server antwortet nicht nur mit einer Seite, sondern auch mit einem Code, der sagt, ob alles gut ging:

Code Bedeutung Beispiel
200 OK Deine Anfrage war erfolgreich ✓
301 Moved Permanently Die Seite wurde woanders hin verschoben
404 Not Found Die Seite existiert nicht ✗
500 Server Error Der Server hat einen Fehler
Wichtig

Die Status-Codes helfen dir zu verstehen, was beim Laden einer Website passiert – sehr wichtig für Debugging!

1.1.6 Übung: Internet-Detektiv

Jetzt wirst du selbst zum Detektiv und schaust, was bei deinen liebsten Websites passiert!

Deine Aufgaben:

  1. Öffne Google Chrome (oder einen anderen Browser)
  2. Drücke F12 oder Rechtsklick → Untersuchen
  3. Gehe zum Tab “Netzwerk” (Network)
  4. Besuche eine Website, z.B. www.example.com
  5. Schau dir die Requests an:
    • Welche Requests wurden gemacht?
    • Was ist der Status-Code der HTML-Datei?
    • Wie heißt der Server (Host)?
    • Gibt es GET-Parameter in der URL?
  6. Probiere verschiedene Websites aus:
    • Google (mit Suche)
    • YouTube (klick auf ein Video)
    • Deine Lieblingswebsite

Fragen zum Nachdenken: - Wie viele Requests werden gemacht, wenn du eine Website aufrufst? - Warum braucht der Browser mehrere Requests? - Welche Status-Codes hast du gefunden?

1.2 UE 2: HTML – Die Sprache der Webseiten

HTML (HyperText Markup Language) ist die Sprache, in der Webseiten geschrieben werden. HTML nutzt Tags (Befehle), um dem Browser zu sagen: “Das ist eine Überschrift”, “Das ist ein Bild”, usw.

1.2.1 HTML-Grundstruktur

Jede HTML-Datei sieht so aus:

<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Das ist der Tab-Name</title>
  </head>
  <body>
    <!-- Hier kommt der sichtbare Inhalt -->
    <h1>Hallo Welt!</h1>
  </body>
</html>
  • <!DOCTYPE html>: Sagt dem Browser: “Das ist HTML5”
  • <html>: Das äußerste Tag
  • <head>: Metadaten, nicht sichtbar
  • <body>: Der sichtbare Inhalt

1.2.2 Die wichtigsten Tags

Tag Verwendung Beispiel
<h1> bis <h6> Überschriften <h1>Großer Titel</h1>
<p> Absatz <p>Ein langer Text...</p>
<ul> / <ol> Ungeordnete/Geordnete Liste <ul><li>Punkt 1</li></ul>
<a> Link <a href="www.google.com">Klick mich</a>
<img> Bild <img src="foto.jpg" alt="Mein Foto">
<table> Tabelle <table><tr><td>Zelle</td></tr></table>

1.2.5 HTML-Formulare

Formulare sind super wichtig! Sie sind die Brücke zwischen HTML und Python/Flask. Mit Formularen kannst du Daten vom User sammeln.

Grundstruktur:

<form method="POST" action="/submit">
  <label for="name">Name:</label>
  <input type="text" name="name" id="name">

  <label for="email">Email:</label>
  <input type="email" name="email" id="email">

  <label for="message">Nachricht:</label>
  <textarea name="message" id="message"></textarea>

  <button type="submit">Absenden</button>
</form>
Wichtig

Das name-Attribut ist SUPER WICHTIG! Im Flask-Code kannst du dann mit request.form['name'] auf die Daten zugreifen. Das ist die Brücke zwischen HTML und Python!

Die wichtigsten Input-Typen:

Typ Zweck Beispiel
text Einzeiliger Text <input type="text" name="username">
email Email-Adresse <input type="email" name="mail">
password Passwort (wird verborgen) <input type="password" name="pwd">
number Zahl <input type="number" name="alter">
tel Telefonnummer <input type="tel" name="phone">
date Datum-Picker <input type="date" name="birthday">

1.3 UE 3: CSS und Bootstrap

Bisher haben wir nur graue, langweilige Websites! Mit CSS machen wir sie schön und farbig.

1.3.1 CSS-Grundlagen

CSS (Cascading Style Sheets) ist die “Schminke” für HTML. Mit CSS legst du fest: Farben, Größen, Abstände, Fonts, usw.

Grundstruktur:

selector {
  property: value;
}

Beispiel:

h1 {
  color: blue;           /* Farbe */
  font-size: 48px;       /* Größe */
  text-align: center;    /* Ausrichtung */
  margin-bottom: 20px;   /* Abstand unten */
}

1.3.2 CSS einbinden

CSS bindest du im <head> ein:

<!DOCTYPE html>
<html>
<head>
  <title>Meine Website</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <h1>Hallo!</h1>
</body>
</html>
Hinweis

Speichere dein CSS in einer separaten Datei style.css – das ist professioneller!

1.3.4 Experiment-Phase: Probiere das aus!

1.3.5 Bootstrap – Das Geheimgeheimnis für schnelle Websites

Bootstrap ist ein CSS-Framework – fertige Styles, die du einfach verwenden kannst! Das spart unglaublich viel Zeit.

Bootstrap einbinden:

<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mit Bootstrap</title>
  <!-- Bootstrap CSS -->
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css"
        rel="stylesheet">
</head>
<body>
  <!-- Bootstrap JavaScript -->
  <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
Tipp

Einfach kopieren und in dein HTML einfügen – dann hast du Zugang zu allen Bootstrap-Klassen!
